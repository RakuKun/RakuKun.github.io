COSMOS (ongoing)
COSMOS is a board game I am working on as the course project for Stone Librande's Game Design Fundamentals class. In general, this is a 1v1v1 deck building game with three different camps and unique cards for each.
time spring 2020
genre deck-building board game
work game design, art
Principle
Stone teaches his Game Design Fundamentals class in a way both theoretical and practical. There is quite a few theories I learn from his lecture and I consider them very as important game design principles as they work for basically all kinds of games (board game or digital game).
game's elements
All good games have something in common without which the game just breaks or not as fun. I call them basic elements.
goals
A game cannot run without goals. The ultimate reason to have a goal is that nobody wants to waste time doing something confused without an end. Also, a goal helps define when the game should end, and how should players reach the goal.
start
Start is where and how players begin a game. All games have start naturally, but start is also a key component of game as an imbalanced start might ruin the experience. And start controls the "distance" between itself and the goal, which also needs good tweaking for the good game flow.
obstacle
Obstacle defines the challenge part which separates game design from experience design. A game should always be challenging in some ways to ask players to spend time either thinking or executing, while an experience does not need to. A good obstacle could be a well controlled balance between "skill" factor and "luck" factor.
rules
Rules control how a game goes, and basically it is the wide collection of all the interactive parts: how game response to a decision? What and how can players do to each other? How should the game end? These questions should all be answered in rules.
Ideation
Since I am a big fan of Slay the Spire, I decide to make a competitive deck-building game. I start my design by studying some board games of the same genre. Learning from those titles, I am able to construct the basic framework of my board game.
research
Slay the Spire
Slay the Spire is a deck-building roguelike made by Mega Crit. A single player game, Slay the Spire features procedurally generated maps and semi-randomly dropped cards and power-ups as the base of its replayability. Its four predetermined characters can their unique deck inspires me of how to create card synergy and balance cards: to create a standard mathematical model for the cost and effect, then add more uncertainty and variety.
Star Realms
Star Realms is a deck-building board game pitting players against each other in a fight for dominance over the entire galaxy. As a multiplayer card game with four Factions (camps), Star Realms does a good job designing the deck-building mechanism in offering a shared trade row which is fair for both sides. In this fast paced game the deck grows stronger in a significant quick way, so learning to balance the early game and late game is also a lesson for me.
outlining
backstory
A star in a faraway galaxy is to explode in supernova soon, which will kill every living thing in its range. Three civilizations begin the exodus:
red civilization are aggressive and always want to wipe out other civilization in the galaxy;
yellow civilization are successful colonizing and work closely among multiple stars;
blue civilization never lack scientists and philosophers because they are so eager to learn new knowledge.
basic structure
COSMOS is a 1v1v1 deck-building turn-bases card game, but different from most games where everyone has "health" and needs to decrease other player's health, in this game each player takes turns to get and play cards to get to the destination on the map.
game components
The cards are in 3 colors: blue, red and yellow, each standing for one civilization. Each civilization has their unique cards that suits its backstory. Each player has a Mothership which needs to be carried to the destination. There are 3 basic collections of ships:
Explorers: The collection of ships mainly for getting resources on the map.
Fighters: The collection of ships mainly for deteriorating the power of other camps, for instance, move their Motherships back or destroy ships.
Boosters: The collection of ships mainly for moving Mothership. Below is the first draft of the game map. The map is divided into many blocks, and all players have the same start point and destination while each other block has a number in it which means the resources.
Iteration
Iteration is absolutely a key faction in game design, and even more crucial for board games as it's not hard to make prototypes but I do need to confirm that if the game has a good flow and make players fully engaged. During the course I will have 2 classes dedicated for playtesting and I also hold off-class playtests with individuals. I keep a record of what I did well and what should improve from each playtester's opinion.
first prototype
rules
The first version is a 1v1 game with a map where each player owns a Mothership that needs to be moved from the start point to the destination. The map has 50 blocks so in most circumstances the Mothership needs to move 50 blocks.
There are two kinds of units: Ships are ones that can move by itself (usually determined by a 6-sided die), and Boosters are ones attached to the Mothership but can make the Mothership move.
Some ships can gain money from the map (indicated by the numbers in blocks) and buy new units from a shared trade list.
problems
The first prototype only has two camps and a relatively limited number of cards, despite of which I still managed to figure out a bunch of problems on balance, game feeling and map.
Balance: It turns out some cards are OP while some are almost useless. Balancing cards is simple as the major work is tweaking the value of each card. But when it comes related to feeling and pace, changing card becomes much harder.
Game feeling: I put card's synergy and their unique abilities on high priority. During playtest, some says they really like the creative design of cards, but other also say that it is hard to understand the rules and playing too many cards in a single turn takes time and energy. It is especially true in late games as the rules allow triggering every ability in a single turn which can cause chaos and difficulty.
Pace: The game feels too slow but once someone gets a key card it can get ridiculously fast. Evidently some cards are OP.
next step

Make a rulebook so I don’t recall rules.
Make a REALLY BIG (and fancy) map.
Design more cards and do 1v1v1 version playtests, if it turns out three people are too messy, just make it a 1v1 game.

Buff or nerf some cards but pay more to more attention to card synergy.
