Third Person Stealth
From the experience of several stealth games I want to analyze what makes them fun and try making a brief stealth level by myself.
duration ~ 6 minutes
theme stealth
gameplay 3D third person
element analysis
Just like other games, there are some fundamental elements in stealth games: a start, a goal, game rules, and obstacles.
start
In stealth games, usually, the start point is a relatively safe place distant from destination. No challenges at the beginning means players have sufficient time to get prepared such as grabbing some equipments and inspecting the level to make a plan. To make the game more challenging, the players may not be able to come back once they set off, so each run needs to be well prepared.
goal
The goal in stealth game is very diverse but has something in common: mostly some fixed points (treasure, exit, important person, etc.). Some games would ask the players to escape once they reach the goal (kill a person probably), and the game pace could change a lot in the escape phase, which adds excitement and up-n-downs to the emotion flow.
rules
The principle of designing rules for stealth games is never to make the playable character too strong that they can even confront a group of armed enemies. Giving ability to engage is dangerous in the sense that a stealth experience may turn into Call of Duty. As a compensation, the character is offered some skills or items so they can pass through enemies unnoticed.
obstacles
There are usually a group of enemies either in fixed points or wandering certain routes, trying to find and catch the playable character. They outnumber the character so it is hard once got into combat, but for a playable level their actions must be predictable so the players can have strategy and plan ahead.
pitfalls
Here are some pitfalls I find before or after playtesting my level.
god path
The god path is a route by which the character have zero chance of being noticed. It usually happens when all enemies' patrol routes don't overlap with the character's route. This bad design could make the level very boring as the players don't need to think, they just go and win.
unpredictable AI
Unpredictable AI is that pre-scripted enemies don't follow a fixed path, or have sudden turn back, or moves too fast. Unpredictable AI makes the game hard in an annoying not challenging way, as the random events would kill player's plan and mess up the level.
forced gameplay
Many stealth games let playable character have items or weapons. This is nice as the players are granted a variety of ways to beat the level. But this could also be a pain when some items are designed very specifically for one level or even one point in level, and the players cannot beat the level if they don't have that. It is always better to provide possibility for trying new ways to explore than limiting the gameplay to one answer.
fundamental rules
It is always good to determine the basic mechanics and components that will be used in the level before blocking it out.
interactions
The player controls a character that basically can walk (at a constant speed), crouch, do interaction with map objectives.
components
All enemies have some attributes including speed and vision range. Since the level has a stealth theme, it will have walls, doors, stairs and map objectives.
core mechanism
Core mechanism will cover the stealth game's basic rules and gameplay, including the action set the character has, enemy types and most importantly, the goal and walkthrough of the game.
character's actions
-Character can walk and run (2 times the speed of walking).
-Character can knock down enemies in melee range.
-Character will have to interact with the trigger item.
enemy AI
-Enemies have fixed patrol, mostly continuous with some stopping over time.
-Enemies have fixed vision range.
-Enemies will engage when they spot player's character.
-Enemies will be alerted when they find a body.
goal
-The game consists of two sequential goals.
-The first goal is to get to the destination and trigger the mission item. The second goal is to eliminate an important target silently.
- Two goals are using the same map, but with different scenes and dealing with different enemy AI.
After deciding the core gameplay, I could begin sketching the level in 2D illustration which determines the basic layout and possible routes for players.
2D Layout
At pre-engine phase I use Adobe Illustrator to block out the level and use some legends for the playable character, enemies and goal. I would like to make 2 major scenes in the game: the outdoor scene and the indoor scene. The first goal only requires actions outdoors while the second goal needs the character intrude the house and stealth in a more dangerous situation.
Third Person Story Driven
This is a level specifically developed for exploring the third person experience driven by a story. The design is currently in early ideation phase.
duration ~ 5 minutes
theme exploration, story driven
genre third person action
element analysis
I consider it very important to analyze the level element and how it connects the story before sketching.
start
In this level, the playable character is spawned at a place where they can have a overview of the first part of the level and are able to draw a very rough sketch of routes they will be taking. Compared to stealth level, there is more unknown at the beginning, leaving room for a surprising story.
goal
The first and told goal in this level is to reach a space where treasure locates. After getting the treasure, the place start to collapse and the character needs to escape which means to get to another place. I try this two-step goal to see how level and goal influences player's emotion flow.
rules
In the planned early demo, the only actions playable character can do is running and jumping. The basic movement allows players to get to most places they see, and it is easier for me to do layout as I don't need to consider the interaction between enemies for now.
obstacles
Traditionally games would put puzzles before treasure so the player would feel very satisfied to get the treasure. In my case, the puzzle is replaced with a long path and some puzzle-like mechanism. After getting the treasure, the scene will change and the character need to escape where they are within limited time.
layout
2D layout
I start design doing 2D layout as it is the fastest way for iteration and prototyping. The 2D sketch mainly shows the path and possible enemies and items. Up and downs in terrain is marked with different color.
3D layout
Once the basic level structure and narrative element is determined, I will start doing 3D layout. The content is to be added soon!
